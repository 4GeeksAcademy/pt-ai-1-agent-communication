# Class Cadence

## Purpose
Define realistic delivery pace for classroom development constraints.

## Working Rhythm
- Class schedule: 3 days per week.
- Effective development time: 60 to 90 minutes per class day.
- Preferred planning unit: 2-week blocks (6 sessions).

## Planning Rules
- End each session with one demoable result.
- Prefer vertical slices over subsystem-only work.
- Keep session scope small enough to complete within one class period.
- Defer non-critical polish if it blocks demo readiness.

## Session Template
- 10 to 15 min: recap and define session goal.
- 35 to 55 min: implementation work.
- 10 to 20 min: validation, demo check, and notes in `devlogs/`.

## Definition of Done Per Session
- Feature increment works in a local demo flow.
- Any required fallback behavior is implemented or clearly documented.
- Risks and next-step tasks are recorded in `devlogs/`.

## Related Documents
- `plan.md` for sprint and phase execution details.
- `decisions.md` for constraints and architectural choices that affect scope.
